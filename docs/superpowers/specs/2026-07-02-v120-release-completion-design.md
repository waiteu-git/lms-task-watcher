# v1.2.0 本格開発 着手設計（リリース完了 + 付加価値機能の分割）

## 背景

`develop`ブランチでv1.2.0（サブスクリプション基盤）の実装はほぼ完了している（決済・Webhook・プレミアムUI・E2Eの主要フローは動作確認済み）。残っているのは運用面のクローズ作業のみで、新規の設計判断は不要。

一方で`TASKS.md`にはv1.2.0の次段階（付加価値機能）の項目がなく、`project_subscription_plan.md`メモリに優先順位（①Discord招待 ②カスタム通知ルール ③統計 ④スヌーズ）が記載されているのみ。

規模が大きいため、本セッションでは2つのサブプロジェクトに分割する。

- **Phase A（本設計の対象）**: v1.2.0リリースを本番稼働させるための残タスク（ストア申請は含まない）
- **Phase B（今後、個別にブレインストーミング）**: 付加価値機能をDiscord→カスタム通知ルール→統計→スヌーズの順に着手
- **Phase C**: Phase B完了後、Chrome Web Storeへv1.2.0を申請

Discordの招待機能は当初「フェーズ2リリースと同時」の予定だったが、今回はPhase Aに含めず後回しにすることをユーザーと確認済み。また、ストア申請はPhase B（付加価値機能）実装後に行う方針とし、Phase Aの完了条件からも外した。付加価値機能を含んだ状態で一度に申請するため。

## Phase A: v1.2.0リリース残タスク

トレードオフのある設計判断はなく、実行順序と役割分担のみを定義する。

1. **ラズパイAPI変更のコミット・push**
   - 対象: `api/routes/webhook.js`（`current_period_end`のnullチェック、`checkout.session.completed`のasync化とStripe再取得によるperiod_end即時保存）、`api/server.js`（`/checkout-success`・`/checkout-cancel`ルート追加）、`api/package-lock.json`
   - 差分は現状の未コミット変更と一致することを確認済み（2026-07-02時点）
   - 実行: Claude Code（SSH経由）

2. **テスト決済の再実行と検証**
   - Stripeテストモードで決済を行い、`current_period_end`がDBおよび拡張機能のプレミアム設定パネルに正しく反映されることを確認する
   - 決済フォームへのカード情報入力はユーザーが実施（Claude Codeはブラウザ決済操作を行わない）
   - DB確認・APIログ確認・拡張機能側の表示確認はClaude Codeが担当

3. **v1.2.0フルフローE2Eテスト**
   - 未登録 → 登録 → ログイン → チェックアウト → Webhook受信 → サブスク有効化 → プレミアム機能利用、の一連を確認する

4. **本番モードへの切り替え**
   - `bash ~/pm2-env.sh prod`を実行し、本番Stripeキーへ切り替える
   - 実行前にユーザーへ確認を取ってからClaude CodeがSSH経由で実行する

## Phase B: 付加価値機能（概要のみ、個別に設計する）

優先順位は`project_subscription_plan.md`メモリの記載を踏襲する。各機能は着手時に個別のブレインストーミング（設計）サイクルを回す。本設計では詳細化しない。

1. 限定Discordサーバー招待
2. カスタム通知ルール（科目・曜日ごとのタイミング設定）
3. 統計・振り返り機能（提出率・提出タイミング傾向）※LETUSから提出時刻が取得可能か事前調査が必要
4. スヌーズ/後でやるリスト

## 完了の定義（Phase A）

- ラズパイのAPI変更がリポジトリにコミット・push済み
- 本番Stripeキーでの決済が実際に成功し、`current_period_end`が正しく表示される

## 完了の定義（Phase C、将来）

- Chrome Web Storeにv1.2.0（付加価値機能込み）が申請済み（審査待ち状態になる）
