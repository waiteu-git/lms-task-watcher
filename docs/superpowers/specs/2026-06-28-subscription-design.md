# LETUS Task Watcher — サブスクリプション機能 設計書

作成日: 2026-06-28

---

## 概要

Chrome拡張機能「LETUS Task Watcher」にサブスクライブ型の有料機能を追加する。
既存の無料機能はそのまま維持し、追加機能型で有料プランを展開する。

---

## フェーズ構成

| フェーズ | 内容 | タイミング |
|---------|------|-----------|
| フェーズ1 | 現状の無料機能でChrome Web Storeに公開・ユーザーを集める | 審査通過後すぐ |
| フェーズ2 | サブスク基盤構築 + プレミアム機能3種リリース | フェーズ1の数ヶ月後 |
| フェーズ3 | React Nativeモバイルアプリ + プッシュ通知 + データ同期 | フェーズ2の安定後 |

---

## 有料機能一覧

### フェーズ2（サブスク解禁時）

- 課題の優先度・メモ付け
- テーマ変更
- サブスクライバーバッジ表示

### フェーズ3（モバイルアプリ連携時）

- React Native製モバイルアプリへのプッシュ通知（iOS / Android）
- Chrome拡張 ↔ スマホ間のデータ同期

### 除外した機能

- Slack / LINE / Discord 通知転送（モバイルアプリ通知に一本化するため不要）

## セクション1: 全体アーキテクチャ

```
[Chrome拡張機能]
  ├── 無料機能（ログイン不要）
  │     └── 現状の全機能（課題スキャン・通知・ダッシュボード等）
  └── 有料機能（ログイン必要）
        ├── 課題の優先度・メモ付け
        ├── テーマ変更
        └── サブスクライバーバッジ表示
              ↕ HTTPS（Cloudflare Tunnel経由）
[ラズパイ4 — Node.js APIサーバー]
  ├── ユーザー認証（メール + パスワード / JWT）
  ├── サブスク状態管理（SQLite）
  └── Stripe Webhook受信 → サブスク状態を更新
              ↕
[Stripe] — 決済・サブスク管理

[Cloudflare Tunnel] — ラズパイをインターネットに安全に公開（無料）
[独自ドメイン] — api.your-domain.com 等（年1,000〜2,000円程度）
```

## セクション2: バックエンド構成（ラズパイ4）

### ディレクトリ構成

```
api/
├── routes/
│   ├── auth.js           # 登録・ログイン・JWT発行
│   ├── subscription.js   # サブスク状態確認
│   ├── user.js           # メモ・優先度・テーマ設定の保存・同期
│   ├── webhook.js        # Stripe Webhookの受信
│   └── device.js         # （フェーズ3用の口のみ）モバイルプッシュトークン登録
├── db/
│   └── sqlite.js         # SQLite操作
└── middleware/
    └── auth.js           # JWT検証ミドルウェア
```

### SQLiteテーブル設計

| テーブル | 主なカラム |
|---------|-----------|
| `users` | id, email, password_hash, created_at |
| `subscriptions` | user_id, stripe_customer_id, stripe_subscription_id, status, current_period_end |
| `user_data` | user_id, assignment_id, priority, memo, updated_at |
| `user_settings` | user_id, theme, updated_at |
| `devices` | user_id, push_token, platform, created_at（フェーズ3用） |

### 負荷軽減方針

- JWTをサーバーレスで検証（ログイン後はトークンをローカルで確認）
- サーバーへの問い合わせは「ログイン時」「サブスク状態確認時（1日1回）」のみ
- SQLiteはファイルベースで別プロセス不要

## セクション3: Chrome拡張機能側の変更

### 認証UIの方針

- 有料機能を使おうとした時だけログインUIを表示
- 通常使用にアカウント作成・ログインは一切不要

```
ダッシュボード
├── 通常の課題リスト（ログイン不要）
├── [プレミアム] 優先度・メモ入力欄
│     → 未ログイン時:        「サブスクライバー限定機能です」+ ログインボタン
│     → ログイン済み・未サブスク: 「サブスクしてこの機能を使う」+ Stripeへのリンク
│     → サブスク有効:        そのまま使える
└── [プレミアム] テーマ変更・バッジ表示
      → 同上
```

### chrome.storage.local に追加するキー

| キー | 内容 |
|------|------|
| `authToken` | JWTトークン |
| `authTokenExpiresAt` | トークン有効期限（ISO） |
| `subscriptionStatus` | `active` / `inactive` / `null` |
| `subscriptionCheckedAt` | 最終確認日時 |
| `subscriptionGraceUntil` | グレース期間終了日時 |
| `assignmentMemos` | `{ [assignmentId]: { priority, memo } }` |
| `assignmentMemosSyncedAt` | 最終同期日時 |
| `theme` | `default` / `dark` / その他 |

## セクション4: Stripe連携フロー

```
1. ユーザーが「サブスク登録」ボタンをクリック
2. 拡張機能がサーバーに「Stripeチェックアウトセッション作成」をリクエスト
3. サーバーがStripe APIを叩いてチェックアウトURLを生成
4. 拡張機能がそのURLをchromeタブで開く（拡張機能内で決済画面は開けないため）
5. ユーザーがStripeの決済画面でカード情報入力・支払い
6. StripeがラズパイのWebhookエンドポイントに「支払い完了」を通知
7. サーバーがSQLiteのサブスク状態を active に更新
8. 拡張機能が次回のサブスク確認時にサーバーから active を受け取り、機能解放
```

## セクション5: ネットワーク障害時のUX保護

### 基本方針: ローカルファースト、サーバーはあくまで補助

サブスクライバーが「お金を払っているのに機能が使えない」状況を絶対に避ける。

### サブスク状態のグレース期間

| 状態 | ユーザー表示 | 機能 |
|------|------------|------|
| サーバー正常 + active | なし（透過的） | フル機能 |
| サーバー障害 + キャッシュ有効（7日以内） | なし（透過的） | フル機能 |
| サーバー障害 + キャッシュ期限切れ（+3日猶予） | 小さな警告バナーのみ | フル機能維持 |
| 10日超過 + 未解決 | 「確認できませんでした」 | 機能制限 |

### プレミアムデータのローカルファースト保存

- メモ・優先度・テーマは即座に `chrome.storage.local` に保存（サーバー待ち不要）
- バックグラウンドでサーバーに同期を試みる
- 同期失敗時は「未同期フラグ」を立てて次回再試行
- 常にローカルから読む（同期状態をユーザーに意識させない）

### JWTトークン管理

- 有効期限: 30日
- 期限7日前にバックグラウンドで自動更新
- 更新失敗時はサイレントリトライ
- 完全期限切れ + サーバー障害 → さらに3日グレース延長後に再ログイン要求

### エラーメッセージの出し分け

| 原因 | 表示内容 |
|------|---------|
| ネットワークエラー | 「サーバーに接続できません（キャッシュで動作中）」 |
| 未サブスク | 「この機能はサブスクライバー限定です」 |
| トークン期限切れ | 「再ログインが必要です」 |

## セクション6: インフラ（ラズパイ4セットアップ）

### 必要部品・費用

| 品目 | 目安費用 | 備考 |
|------|---------|------|
| USB-SSD（240GB程度） | 3,000〜5,000円 | SDカードは24h稼働で破損リスクあり |
| 公式電源アダプタ（5V/3A USB-C） | 1,000〜1,500円 | 電力不足はデータ破損の原因 |
| ケース（ファン付き） | 1,500〜3,000円 | 常時稼働の放熱対策 |
| LANケーブル | 500円程度 | サーバー用途は有線必須 |
| UPS（無停電電源装置） | 5,000〜10,000円 | 瞬断によるSQLite破損防止、推奨 |
| 独自ドメイン | 年1,000〜2,000円 | Cloudflare Tunnel公開に必要 |

- 初期費用概算: UPSあり 11,000〜20,000円 / UPSなし 6,000〜10,000円
- ランニングコスト: ドメイン年1,000〜2,000円 + 電気代月100〜150円

### ソフトウェアスタック

- OS: Raspberry Pi OS Lite（64bit）
- ランタイム: Node.js LTS
- 外部公開: Cloudflare Tunnel（無料）
- SSL: Cloudflare経由で自動（証明書不要）
- DB: SQLite（`better-sqlite3`）
- プロセス管理: PM2（クラッシュ時自動再起動）

## セクション7: Chrome Web Store審査への対応

フェーズ2でバージョンアップ申請時に審査で問われる変更点：

| 変更内容 | 審査リスク | 対応 |
|---------|-----------|------|

| ログインUI追加 | 低 | 説明文にオプション機能であることを明記 |
| テーマ変更・バッジ | 低 | 特に対応不要 |
