# LETUS Task Watcher — 手動課題追加・βオプトイン・課題共有 構想

作成日: 2026-06-28  
ステータス: **構想段階（サブスク基盤フェーズ2実装後に着手）**

---

## 概要

サブスク基盤（フェーズ2）の安定後に追加する3つの機能群の設計構想。

1. **βオプトイン機能** — サブスクライバーが新機能を早期試用できる仕組み
2. **手動課題追加（無料）** — LETUSが自動検出できない課題（フォーラム投稿型等）を自分で登録
3. **サブスクライバー間課題共有（βとして第1弾）** — 同コースのサブスクライバーが追加した手動課題を取り込める

---

## フェーズ位置づけ

| フェーズ | 内容 |
|---------|------|
| フェーズ2 | サブスク基盤（メモ・優先度・テーマ・バッジ） |
| フェーズ2.5 | 本設計の機能群を段階的に追加 |

## 機能1: βオプトイン（ハイブリッド方式）

### 仕組み

```
server: { beta_features: ["manual_tasks", ...] }   ← 開発者がオン/オフ制御
local:  { betaOptIn: true }                         ← ユーザーの参加意思
→ 両方 true のときだけ機能が有効
```

### 設計方針

- **サーバーが「どの機能がβ中か」のリストを定義する**（開発者による緊急停止・全体ロールバックが可能）
- **ユーザーの参加意思（ON/OFF）は `chrome.storage.local` に保存**（サーバー障害時もキャッシュで動作継続）
- βリストはスキャン時にサーバーから取得してキャッシュする
- サブスクライバーのみがオプトイン可能（非サブスクにはUI非表示）

### サーバー側追加

- `GET /api/beta/features` — 現在βとして公開中の機能リストを返す
- `user_settings` テーブルに `beta_opt_in: boolean` カラム追加

### chrome.storage.local 追加キー

| キー | 内容 |
|------|------|
| `betaOptIn` | ユーザーのβ参加意思（boolean） |
| `betaFeatures` | サーバーから取得したβ機能リスト（配列） |
| `betaFeaturesFetchedAt` | 最終取得日時 |

## 機能2: 手動課題追加（無料機能）

### UI: LETUSページ上のフローティングボタン

- Content Script（`public/content.js`）で実装
- **通常時は格納されたボタン**のみ表示（画面右下固定）
- **ボタンクリックでフォームが展開**
- LETUSのページUIを改変しない（shadow DOMで隔離推奨）

### 手動追加の対象範囲

LETUS上のあらゆるページで手動追加を許容する。自動スキャンが拾えない課題の形式は多岐にわたるため、URLパターンで表示を絞り込まない。

想定される追加パターン（例）：
- フォーラムへの投稿課題
- 提出BOXがある課題（補足情報として手動登録したい場合）
- PDFが掲載されており教員へ直接提出する課題
- LETUS上にページはあるがスキャンで拾えていない課題

現在のページURLを LETUS URL として自動セットするため、どのページから追加しても適切なリンクが付与される。

### 格納ボタンのデザイン方針（通常状態）

- 小さなアイコンボタン（32×32px程度）
- LETUS上の既存リンクやボタンと被らない位置（右下固定）
- ホバー時のみラベル（「課題を追加」）を表示
- **表示対象: `https://letus.ed.tus.ac.jp/*` 全ページ**（絞り込みなし）

### スキャン済み課題ページでのUI切り替え（将来、無料版も対象）

現在開いているLETUSページに対応する課題がすでにスキャン済みで登録されている場合、格納ボタンの代わりに**ステータスインジケーター**を表示する。

```
┌──────────────────────────┐
│ ✓ 登録済み               │
│ 締切: 7/5 23:59          │
└──────────────────────────┘
```

- 課題名・締切日時を表示（クリックでダッシュボードの該当課題にジャンプ）
- 「手動追加」ボタンは非表示（重複追加を防ぐ）
- **無料版ユーザーにも公開可能**（サーバー不要・ローカルデータのみで実現）
- 実装タイミング: 手動課題追加UIと同時、またはその後

### フォーム入力項目

| 項目 | 必須 | デフォルト |
|------|------|-----------|
| 課題名 | ✓ | — |
| 締切日時 | ✓ | — |
| コース | ✓ | 現在のURLから自動検出（`/course/view.php?id=XXXX`） |
| LETUS URL | 任意 | 現在のページURLを自動セット |
| メモ | 任意 | — |
| 優先度 | 任意 | サブスクライバーのみ保存（非サブスクは入力欄非表示） |

### データ保存先

- `chrome.storage.local` の `manualAssignments` キーに保存
- 自動スキャン課題と同じリストに混在して表示（出典フラグで区別）
- 削除・編集はダッシュボードから可能

### 表示上の区別

- 手動追加課題には「手動」バッジを付与（自動スキャン課題と視覚的に区別）

## 機能3: サブスクライバー間課題共有（βフェーズ第1弾）

### 対象

- サブスクライバーかつβオプトインONのユーザー

### フロー

```
① ユーザーが手動課題を追加する際「共有する」オプションをONにする
② サーバーの shared_tasks テーブルに登録される
③ 同コースの他のサブスクライバーのスキャン時に共有課題リストを取得
④ 「○件のクラスメート共有課題があります」と通知
⑤ ダッシュボードの「クラスメートからの共有」セクションで内容を確認
⑥ 取り込む課題を選択して承認 → 自分の manualAssignments に追加
```

### 取り込み後の表示・挙動

共有課題を取り込んだ後は、**自分がその内容を手動追加したときと完全に同じ扱い**になる。元の投稿者が誰かは取り込み後に関与しない。

LETUSのURLの有無によって2パターンに分かれる：

| パターン | 条件 | 取り込み後の扱い |
|---------|------|----------------|
| **LETUSリンクあり** | フォーラムや提出BOXのURLが付与されている | LETUS上のURLを持つ手動課題として扱う。課題カードからURLを開ける。自動スキャン課題と同等のリンク動作 |
| **LETUSリンクなし** | 教員への直接提出など、LETUS上に提出箱が存在しない | URLなし手動課題として扱う。カードにリンクは表示されない。メモ・締切のみで管理 |

取り込み時に元の `letus_url` フィールドをそのまま引き継ぐことで、この2パターンを自然に実現する（分岐ロジック不要）。

### 重複対策（B案 + 警告補助）

- 取り込む前に候補一覧を表示（どれを選ぶかはユーザー判断）
- **同じコース・同じ日付の課題が既に共有されている場合、追加時に警告を表示**（「同じ締切の課題が既に○件あります」）
- 自動マージは行わない（誤合算を防ぐ）

### サーバー側追加

#### テーブル: `shared_tasks`

| カラム | 型 | 内容 |
|--------|-----|------|
| id | INTEGER PK | — |
| user_id | INTEGER FK | 投稿者 |
| course_id | TEXT | LETUSコースID |
| title | TEXT | 課題名 |
| due_at | TEXT | 締切（ISO） |
| letus_url | TEXT | 任意 |
| memo | TEXT | 任意（共有メモ） |
| created_at | TEXT | — |

#### エンドポイント

| メソッド | パス | 内容 |
|---------|------|------|
| `POST` | `/api/shared-tasks` | 共有課題を投稿 |
| `GET` | `/api/shared-tasks?courseIds=...` | 指定コースの共有課題一覧取得 |
| `DELETE` | `/api/shared-tasks/:id` | 自分が投稿した共有課題を削除 |

### プライバシー考慮

- 共有課題の投稿者名は他のユーザーに表示しない（匿名）
- 自分が追加した課題のみ削除可能
