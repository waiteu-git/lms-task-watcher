## 機能1: changelog.html（更新通知ページ）

### トリガー

`src/background/index.ts` の `chrome.runtime.onInstalled` リスナーで `reason === 'update'` のときのみ `changelog.html` を新規タブで開く。

```ts
chrome.runtime.onInstalled.addListener((details) => {
  // 既存: アラーム登録
  chrome.alarms.create(...)

  if (details.reason === 'update') {
    void chrome.tabs.create({ url: chrome.runtime.getURL('changelog.html') })
  }
})
```

### ページ構成

- 配置: `public/changelog.html`（ビルド時に `dist/` へコピー）
- 内容: バージョン番号（ハードコード）・変更点リスト・高レベルロードマップ
- スタイル: インラインCSS、最大幅640px・中央寄せ

### バージョン管理

リリースごとに `public/manifest.json` のバージョンと `public/changelog.html` の内容を手動で更新する。

## 機能2: 初回オンボーディングバナー

### 概要

インストール直後のユーザーがセットアップ手順に迷わないよう、ポップアップの最上部にステップバナーを表示する。初回スキャン完了後に永久に非表示になる。

### 状態フラグ

`chrome.storage.local` に `onboardingCompleted: true` を保存する。

- 未設定（デフォルト）: バナーを表示
- `true`: バナーを表示しない（永久）
- フラグを書き込むタイミング: `updateNow()` が初めて正常完了したとき

### 表示ステップ

ポップアップマウント時に現在のデータ状態を判定し、対応するステップを表示する。

| ステップ | 表示条件 | 表示内容 |
|---------|---------|---------|
| Step 1/3 | 登録済みコースが0件 | 「まずLETUSにアクセスしてください。コースページを開くと自動で登録されます。」＋[LETUSを開く]ボタン |
| Step 2/3 | コース登録済み・有効化済みが0件 | 「ダッシュボードを開いて、追跡したいコースにチェックを入れてください。」＋[ダッシュボードを開く]ボタン |
| Step 3/3 | 有効化済みコースあり・スキャン未実施 | 「準備完了！「今すぐ更新」を押して課題を取得してください。」 |

スキャン未実施の判定: `lastRefreshAt === null`

### 消滅タイミング

`updateNow()` の `finally` ブロックより前、正常完了を確認した直後に `onboardingCompleted: true` を書き込む。これによりバナーは再表示されない。

エラー終了時・ログイン未了での早期リターン時はフラグを書き込まない（再度ポップアップを開いたときにバナーが継続表示される）。

### UI仕様

- ポップアップ最上部に固定表示（既存コンテンツの上）
- × ボタンなし（スキャン完了以外では消えない）
- ステップ番号・アイコン・説明文・アクションボタン（ステップ1・2のみ）

### 実装ファイル

| ファイル | 変更種別 | 内容 |
|---------|---------|------|
| `src/background/storageKeys.ts` | 追加 | `ONBOARDING_COMPLETED_KEY` 定数を追加 |
| `src/components/OnboardingBanner.tsx` | 新規 | ステップ判定・表示・ボタン処理 |
| `src/App.tsx` | 変更 | バナーのマウント、`updateNow()` 完了後にフラグ書き込み |

`background/index.ts` は変更不要。フラグの初期化は「未設定＝初回」として扱うため明示的なセットアップは不要。

## 対象外

- × ボタンによる手動閉じ（スキャン完了のみが消滅条件）
- アップデート時のオンボーディング再表示（インストール時のみ）
- ダッシュボードへのバナー表示（ポップアップのみ）
