# v1.1.x QA・修正設計（手動課題追加機能のリリース準備）

## 背景

`release/v1.1.x` には手動課題追加機能（コミット `a748924` 完了済み）が実装されている。
この機能を Chrome Web Store にリリースする前に、コードベースを精査し見つかった不具合・設計上の不整合を解消する。

- 対象ブランチ: `release/v1.1.x` から分岐した `qa/v1.1.x-release`
- `main`（v1.0.x ストア公開版）・`develop`（v1.2.0 サブスク開発）には影響しない

## 現状の実装

- `src/content/manualTaskWidget.ts`: LETUS全ページに右下フローティングウィジェットを表示。コースが1件でも登録されていれば、課題提出ページでスキャン済み課題ならインジケーター、それ以外なら手動追加フォームの `+` ボタンを表示
- `src/core/manualAssignment.ts`: `ManualAssignment` 型と CRUD（`getManualAssignments` / `saveManualAssignments` / `addManualAssignment` / `deleteManualAssignment`）
- `src/components/ManualAssignmentSection.tsx`: ダッシュボードの末尾に独立セクションとして固定表示。削除ボタンのみ（Undo無し）
- 通知（`App.tsx` の `checkDeadlineWarningNotifications`、`src/background/index.ts` の `runAutoScan`→`notifyDeadlineSummary`）はLETUSスキャン課題（`Assignment` 型）のみが対象。手動課題（`ManualAssignment` 型）は通知されない

## 見つかった問題

### バグ
1. `App.tsx` の `resetAllData()` が `MANUAL_ASSIGNMENTS_KEY` を削除対象に含めていない。「すべての保存データを初期化」を実行しても手動課題が残る
2. `manualTaskWidget.ts` のフォーム送信成功後、`wt-title` / `wt-deadline` / `wt-memo` はクリアされるが `wt-course` の `<select>` の選択状態がリセットされない

### 設計上の不整合
3. ウィジェットがLETUSの全ページ（`/my/` トップや科目一覧など）に表示され、文脈に合わない
4. 手動課題が締切通知（1h/3h/24h前）の対象外
5. 手動課題がダッシュボードのタイムライン（24時間以内・明日まで・今週・それ以降）に統合されておらず、独立セクションとして末尾に固定表示される
6. 手動課題の削除が即時・取り消し不可（LETUSスキャン課題の「非表示」にはUndoトーストがある）
7. 手動課題に提出状態（`submitted`）の概念がなく、提出済みでも常に表示され続ける

## 修正方針

### 1. バグ修正

- `resetAllData()` に `MANUAL_ASSIGNMENTS_KEY` の削除を追加する
- フォーム送信成功時に `wt-course` の `selectedIndex` を `0` にリセットする

### 2. ウィジェット表示条件の限定

`initManualTaskWidget()` の入り口に URL パターン判定を追加し、以下のページ以外では何も表示しない。

- コースページ: `/course/view.php?id=XXX`
- 課題提出ページ: `/mod/assign/view.php?id=XXX`

```ts
function isCoursePage(): boolean {
  return location.pathname.includes('/course/view.php')
    && new URLSearchParams(location.search).has('id')
}

function isAssignmentPage(): boolean {
  return location.pathname.includes('/mod/assign/view.php')
    && new URLSearchParams(location.search).has('id')
}
```

### 3. コースページUI

コースページでは2つの要素を表示する。

**(a) 各課題リンクへのインラインバッジ（コンパクト）**

コースページ内の課題リンク（`/mod/assign/view.php?id=XXX` 形式）をスキャンし、それぞれの行末に状態バッジを差し込む。

| 状態 | 表示 | 操作 |
|---|---|---|
| スキャン済み課題と一致 | 締切日時 + `✓`(提出済み) / `○`(未提出) | タップ不可（スキャン結果をそのまま反映） |
| 手動追加済み課題と一致 | 締切日時 + `✓`/`○` + 編集アイコン(`✎`) | `✓`/`○` タップで `submitted` をトグル、`✎` で編集フォームを開く |
| 未登録 | `+` のみ | タップで追加フォームを開く（タイトル・URLは当該リンクから自動入力） |

一致判定は、スキャン済み課題は `Assignment.url`、手動課題は `ManualAssignment.letusUrl` と、リンクの正規化済みURL（`#`以降を除去）で比較する。

バッジは各課題リンクの隣に `inline-block` の小さな `<span>` をShadow DOM経由で挿入する形で実装する（コースページのDOM構造はLETUS側のレイアウトに依存するため、リンク要素の親 `<li>` または直後に挿入する）。

**(b) 右下フローティングウィジェット**

現状の「課題を追加」`+`ボタンと同じものを維持する。コースページに存在しない課題（オフラインで出された課題、対面で伝えられた課題など）を自由に追加できるようにする。コースの `courseId` は現在のURLから算出したものをデフォルト選択する。

### 4. 課題提出ページUI

現状を維持する。スキャン済みならインジケーター、未スキャンなら `+` ボタン。

### 5. ManualAssignment 型の拡張

```ts
export type ManualAssignment = {
  id: string
  courseId: string
  courseName: string
  title: string
  letusUrl: string | null
  deadline: string
  memo: string
  submitted: boolean   // 追加
  createdAt: string
}
```

既存データ（`submitted` フィールド無し）は読み込み時に `submitted: false` として扱うマイグレーション処理を `getManualAssignments()` に追加する。

### 6. ダッシュボードのタイムライン統合

- `ManualAssignmentSection`（独立固定セクション）を廃止する
- 手動課題を締切に基づいて `urgentAssignments` / `tomorrowAssignments` / `thisWeekAssignments` / `laterAssignments` 相当のグルーピングに混在させる
- 手動課題用の表示コンポーネント（現行 `ManualAssignmentSection` 内の `<li>` 相当、「手動」バッジ付き）を各セクション内でLETUSスキャン課題の `AssignmentCard` と並べて表示する
- 提出済み（`submitted: true`）の手動課題は「提出済み・完了」セクションに表示する
- 削除ボタンは維持しつつ、Undo機能を追加する（後述セクション8）

### 7. 通知統合

- `App.tsx` の `checkDeadlineWarningNotifications` に手動課題（`submitted: false` かつ締切が未来）を対象として含める
- `src/background/index.ts` の `runAutoScan` 完了後の通知チェックにも同様に手動課題を含める
- 通知キー（`notifiedDeadlineKeys` の `${id}:1h` 等）は手動課題の `id`（UUID）をそのまま使う。LETUSスキャン課題のIDと衝突しない

### 8. 削除のUndo

- 手動課題削除時に直前に削除したアイテムを一時保持し（`lastDeletedManualAssignment` 相当のstate）、既存の `undoToast` UIで「元に戻す」を表示する
- 「元に戻す」押下で `addManualAssignment` により再追加する
- LETUSスキャン課題の「非表示」とは別の状態変数として管理する（混同しない）

## 影響範囲

- `src/content/manualTaskWidget.ts`: 表示条件・コースページUI・フォームリセットの修正
- `src/core/manualAssignment.ts`: 型に `submitted` 追加、マイグレーション処理
- `src/components/ManualAssignmentSection.tsx`: 廃止、各タイムラインセクション内に統合するコンポーネントへ置き換え
- `src/App.tsx`: タイムライン統合、`resetAllData()` 修正、通知統合、Undo追加
- `src/background/index.ts`: 通知統合
- `src/App.css`: インラインバッジ・コンパクトUIのスタイル追加
