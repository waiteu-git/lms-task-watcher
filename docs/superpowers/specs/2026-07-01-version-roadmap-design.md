# LETUS Task Watcher — バージョンロードマップ設計書

作成日: 2026-07-01

---

## 背景

`TASKS.md` に「フェーズ2.5」という名前が2箇所で異なる意味で使われていた（サブスク付加価値の追加機能群と、データ同期基盤）。また、旧「フェーズ4（時間割連携）」は独立した先行フェーズとして計画されていたが、モバイルアプリのリリース方針上、課題管理機能とセットで初期搭載する方が実態に合う。本設計書でバージョン番号と各フェーズの対応関係を整理し直す。

## リリース順序の方針

- v1.1.0とv1.2.0は独立してリリースする。v1.1.0はベータテスト結果を待って`release/v1.1.x` → `main`へ単独マージ・ストア審査提出。v1.2.0はその後`develop` → `main`へ別途リリース（[[project_branch_strategy]]の3本体制を維持）
- v1.2.x（サブスク付加価値）はマイナーバージョンとして v1.2.0 安定後に順次追加。①Discordのみv1.2.0と同時
- v1.3.0（データ同期基盤）はv1.2.0が本番安定稼働した後に着手
- v2.0.0（モバイルアプリ）はv1.3.0のAPIを前提とするため、v1.3.0完了後に着手

## v1.3.0: データ同期基盤の詳細

旧`TASKS.md`「フェーズ2.5」のタスクを踏襲（内容変更なし、名称のみ更新）。

- バックエンド: `POST /api/assignments`（JWT認証、SQLiteへユーザーID紐付け保存）
- バックエンド: `GET /api/assignments`（モバイル向け読み取りAPI）
- `background/index.ts`: スキャン完了後、ログイン済みならバックエンドへPUSH。未ログイン時は従来通りローカル保存のみ
- `lifecycleStatus: 'new' / 'changed'` 付与ロジックとバッジ表示（`AssignmentCard.tsx`）

## v2.0.0: モバイルアプリの詳細

旧フェーズ3（モバイルアプリ）と旧フェーズ4（時間割連携）を統合し、初期リリースから両機能を搭載する。

**理由:** モバイルアプリをリリースする場合、課題機能に加えて時間割機能を確実に含める方針のため、時間割連携を独立した先行フェーズにする必要はなく、モバイルアプリの初期スコープに含める。

### 課題管理（既存設計を踏襲）

詳細は [2026-06-28-mobile-app-design.md](2026-06-28-mobile-app-design.md) を参照。React Native（Expo）、v1.3.0のAPIでChrome拡張とクロスデバイス同期、Expo Push Notificationsでプッシュ通知転送。

### 時間割連携（新規スコープ、要追加設計）

- CLASSシステムの調査（ドメイン確認、授業IDとLETUSコースIDの対応関係の確認）
- バックエンド: `POST/GET /api/timetable` エンドポイント追加
- モバイルアプリ: 時間割グリッド画面（コマタップで対応するLETUSコース詳細に遷移）
- Chrome拡張への時間割グリッドUI追加は任意（モバイル優先、要望があれば検討）

**未確定事項:** CLASSシステムからの時間割データ取得方法（スクレイピング可否・API有無）は未調査。v2.0.0着手時に [2026-06-28-mobile-app-design.md](2026-06-28-mobile-app-design.md) に時間割連携セクションを追記する形で詳細設計を行う。
