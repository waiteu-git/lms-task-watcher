## フェーズ2.5 — データ同期基盤（フェーズ2安定後）

### 目的
Chrome拡張のスキャン結果をバックエンドに同期し、フェーズ3（モバイルアプリ）の前提を整える。

### バックエンド追加エンドポイント
- `POST /api/assignments` — スキャン完了後にChrome拡張がPUSH
- `GET  /api/assignments` — モバイルアプリが読み取り

### Chrome拡張側変更
- スキャン完了後、ログイン済みの場合のみバックエンドにPOST
- 未ログインでも従来どおりローカル（chrome.storage.local）で動作

### コース変更検知
- `lifecycleStatus: 'new' | 'changed'` の付与ロジックを実装
- 締切日時変更・新規課題出現を検知してバッジ表示・通知

## フェーズ3 — モバイルアプリ（フェーズ2.5安定後 / 未着手）

- React Native（Expo）で iOS / Android アプリ開発
- バックエンドの `GET /api/assignments` からデータ取得（LETUSへの直接アクセス不要）
- Expo Push Notification でプッシュ通知（ブラウザ不要）
- **制約**: Chromeが起動してスキャンしないとデータは更新されない
- App Store（$99/年）・Google Play（$25 一時費用）
